# Ghost Shell — Windows installer

Builds a single `.exe` that takes a fresh Windows machine to a working
Ghost Shell install in one click.

## What the installer does

When the user double-clicks `GhostShellAntySetup.exe`:

1. **Welcome / folder picker** — default `%LOCALAPPDATA%\GhostShell`
2. **Python check** — if `python --version` doesn't return 3.11+, runs the
   bundled Python installer silently with `PrependPath=1`
3. **Copies the project** — everything except git/venv/state files
4. **Creates a venv** at `<install>\.venv` and pip-installs `requirements.txt`
   (cryptography, flask, selenium, ...)
5. **Generates a launcher** — `<install>\bin\ghost_shell.bat` that
   activates the venv and runs `python -m ghost_shell dashboard`
6. **Shortcuts** — Desktop + Start menu entries pointing at the launcher
7. **Optional "Launch now"** checkbox on the final wizard page

Uninstaller is auto-generated by Inno: removes everything in the install
folder including the venv, the SQLite DB, scheduler logs.

## Building (you, the developer)

### One-time setup

1. Install **Inno Setup 6+** — https://jrsoftware.org/isinfo.php
2. Download a Windows Python installer to bundle:
   - https://www.python.org/downloads/windows/ → grab the
     "Windows installer (64-bit)" — file is `python-3.12.x-amd64.exe`
   - Drop it into `installer/deps/`

The `deps/` folder is git-ignored (large binary, frequently updated).
End users never see it directly — the installer extracts + runs it.

### Build

From a Windows shell in the project root:

```cmd
cd installer
build.bat
```

`build.bat` finds Inno's `ISCC.exe`, picks up whatever
`python-3.*-amd64.exe` is in `deps/`, and compiles. Output:

    installer\output\GhostShellAntySetup.exe   (~620 MB — most of it is chrome_win64\)

Distribute that single file. End users only need it.

### Tweaking

Open `ghost_shell_installer.iss` to change:

- **App version** → `#define AppVersion "0.2.0"` near the top
- **Default install folder** → `DefaultDirName={localappdata}\GhostShell`
  (try `{commonpf64}\GhostShell` for system-wide install — needs admin)
- **What's bundled** — `[Files] Excludes:` list. Anything not matching an
  excluded pattern is copied. To leave out the legacy folder add
  `\_legacy\*` (already in the list).
- **Shortcut icon** — drop a `.ico` into `installer/assets/` and uncomment
  the `SetupIconFile=` / `UninstallDisplayIcon=` lines at the top.
- **Bundle a different Python version** — just put the new
  `python-3.X.Y-amd64.exe` into `deps/` and re-run `build.bat`. The
  build script picks it up by glob.

### Skipping Python install when not needed

`NeedsPython()` in the `[Code]` section runs `python --version` at install
time. It accepts 3.11 / 3.12 / 3.13 / 3.14 — anything else triggers the
bundled installer. Edit the conditional list there if you want to widen
or narrow the accepted range.

## Project distribution recommendation

For end users you have **two channels:**

1. **GitHub Releases** — upload `GhostShellAntySetup.exe` as a versioned
   release asset. Clean URL like
   `github.com/thuesdays/goodmed/releases/download/v0.2.0/GhostShellAntySetup.exe`.
   Free, no bandwidth cost on you.
2. **Direct download from your site** — host the .exe wherever. The
   installer is self-contained, no dependency on the repo at runtime.

Source code stays in the git repo (lean, <50 MB after `.gitignore` covers
`chrome_win64/`); the .exe goes through Releases.

## Icon branding for chrome.exe (optional but recommended)

The bundled `chrome.exe` has the default Chromium globe icon by default,
because we don't rebuild Chromium with custom icon resources. To make
the running browser show the **Ghost Shell** icon instead (matching
what users see when running from the dev source tree), the installer
post-processes `chrome.exe` and `chromedriver.exe` with
[rcedit](https://github.com/electron/rcedit) — a tiny MIT-licensed tool
that edits Win32 resources.

### Enable

1. Download the latest `rcedit-x64.exe` from
   <https://github.com/electron/rcedit/releases>
2. Save it as `installer/deps/rcedit.exe` (rename so the path matches
   exactly — that's what the .iss looks for)
3. Rebuild — the installer detects the file and stamps `ghost_shell.ico`
   onto chrome.exe and chromedriver.exe automatically

The build is **idempotent** — if `rcedit.exe` is missing, the icon
stamp step is silently skipped (`Check: HasRcedit` short-circuits the
[Run] entries). Setup still produces a valid installer, just without
icon branding.

### Re-brand without rebuilding

`installer/brand_chrome.bat` lets you re-stamp the icon onto an
already-installed copy without rebuilding the .exe. Useful when you
update `ghost_shell.ico`, or when testing icon changes:

```cmd
cd installer
brand_chrome.bat
```

Defaults to `%LOCALAPPDATA%\GhostShellAnty`. Pass an alternate install
dir as the first argument:

```cmd
brand_chrome.bat "C:\ProgramData\GhostShell"
```

After branding, Windows may show the cached old icon for a few seconds.
Force-refresh with:

```cmd
ie4uinit.exe -ClearIconCache
```

or simply log out and back in.
